## 🎯 Problem Statement

Binary classification of chest X-rays: **NORMAL** vs **PNEUMONIA**.

**Why this dataset?**
- Clinically relevant, real-world images (not synthetic)
- Natural class imbalance (~3:1 PNEUMONIA:NORMAL) that requires deliberate handling
- Available directly from HuggingFace — no external credentials needed
- Non-trivial image variability (different scanners, patient demographics)

---

## 📊 Dataset

| Property | Value |
|---|---|
| Source | [`keremberke/chest-xray-classification`](https://huggingface.co/datasets/keremberke/chest-xray-classification) |
| Total images | ~5,800 |
| Classes | NORMAL, PNEUMONIA |
| Train / Val / Test | ~4,200 / ~600 / ~1,000 |
| Class imbalance | ~3:1 (PNEUMONIA:NORMAL) |

**Handling class imbalance:**
1. `WeightedRandomSampler` in the training DataLoader (balances batches)
2. Class-weighted `CrossEntropyLoss` (penalises errors on minority class more)

## 🤖 Model Choices

### Model 1: SimpleCNN (Baseline)
- 3 convolutional blocks (Conv → BN → ReLU → MaxPool)
- Trained from scratch
- **Why**: Establishes a meaningful lower bound; demonstrates scratch ML ability; very fast on CPU

### Model 2: MobileNetV2 (Strong Model)
- ImageNet pretrained backbone, custom 2-layer FC head
- Gradual fine-tuning: backbone frozen for first N epochs, then unfrozen at reduced LR
- **Why MobileNetV2 over ResNet-50**: ~3.4M vs ~25M params — designed for constrained compute; depthwise separable convolutions are CPU-friendly; proven on medical imaging tasks

### Transfer Learning Justification
Medical X-ray images share low-level features (edges, textures) with ImageNet — pretrained features generalise well even across domains. This is supported by the literature on medical image transfer learning.

## ⚙️ Configuration

All parameters are centralised in `configs/config.yaml`:

```yaml
data:
  dataset_name: "keremberke/chest-xray-classification"
  image_size: 224
  batch_size: 32

model:
  name: "mobilenet_v2"    # or "simple_cnn"
  pretrained: true
  freeze_backbone: true
  unfreeze_after_epoch: 5

training:
  epochs: 15
  learning_rate: 0.001
  weight_decay: 1.0e-4
  patience: 3
  seed: 42
```

## 🚀 Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run EDA
```bash
python data/eda.py
# Plots saved to results/eda/
```

### 3. Train baseline (SimpleCNN)
```bash
python src/train.py --model simple_cnn --epochs 15
```

### 4. Train strong model (MobileNetV2)
```bash
python src/train.py --model mobilenet_v2 --epochs 15
```

### 5. Evaluate
```bash
python src/evaluate.py --model mobilenet_v2 --split test
# Saves: confusion matrix, ROC curve, misclassified grid → results/
```

### 6. Run Gradio app locally
```bash
python app.py
# Open http://localhost:7860
```

### 7. Docker
```bash
docker build -t chest-xray-classifier .
docker run -p 7860:7860 chest-xray-classifier
```

## 🔍 Error Analysis

Common failure modes observed:
- **NORMAL misclassified as PNEUMONIA**: Low-contrast images where lung texture is ambiguous
- **PNEUMONIA misclassified as NORMAL**: Mild / early-stage pneumonia with subtle opacification
- Model is more confident on clear-cut cases; ambiguous images cluster near 0.5 confidence

## 📦 Compute Notes

This project is designed to run **without a GPU**:
- MobileNetV2 with frozen backbone trains ~5 min/epoch on CPU (batch=32)
- SimpleCNN trains ~2 min/epoch on CPU
- Dataset cached locally after first download

**If more compute were available:**
- Scale to EfficientNet-B3 or ResNet-50
- Add 5-fold cross-validation
- Implement Grad-CAM saliency maps for model interpretability
- Explore self-supervised pre-training on unlabelled X-ray data
- Hyperparameter sweep with Optuna or Ray Tune

## 🛠️ Engineering Decisions

| Decision | Choice | Reasoning |
|---|---|---|
| Config management | YAML + argparse | Simple, readable, no extra dependencies |
| Imbalance handling | WeightedSampler + weighted loss | Dual strategy is more robust than either alone |
| Logging | Python `logging` module | Production-appropriate, not `print()` |
| Checkpointing | Save only best val_acc | Saves disk space; prevents saving overfit models |
| Gradual unfreeze | Epoch 5 for backbone | Avoids catastrophic forgetting of pretrained features |
| num_workers=0 | Windows compatibility | `multiprocessing` with `spawn` context causes issues on Windows |
